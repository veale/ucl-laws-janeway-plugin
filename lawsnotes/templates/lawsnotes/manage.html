    Custom JS — what's available?

    The Custom JavaScript snippet runs after lawsnotes.js has finished bootstrapping. The following globals are available:

    • window.lawsnotesSettings — plain-object snapshot of the runtime settings (sidenoteBreakpoint, sidenoteWidth, showDropCaps, hideCiteDropdown, hideEndnoteListWhenSidenotes, logoUrl).
    • window.lawsnotesJournalName — string with the current journal's display name.
    • window.lawsnotesApiUrl — URL of the card-metadata JSON endpoint (path-mode aware).
    • window.lawsnotesShowTooltipForMarker(a) — open a footnote tooltip programmatically.
    • window.lawsnotesOnReady(fn) — register a callback that runs once lawsnotes.js is fully bootstrapped, with a stable API surface object: {settings, showTooltipForMarker, buildSidenotes, teardownSidenotes, reflow}.

    Errors in the snippet are caught and logged to the browser console (look for [lawsnotes custom_js]); a broken patch never breaks the page.
